How to serve a web template with Spring Boot

If you’re building a Spring Boot application, then odds are, you’re building a web app. Web apps have been on the rise for twenty years. And with Spring Boot, it’s easier than EVER to build one!

So let’s get to it!

1 — Getting started

To build ANY Spring Boot application, you’ll want to visit https://start.spring.io. This magical web site lets you grab everything you need to build an app. Pick the following options:

  • Build system: Pick your favorite. I like Maven, but many love Gradle.
  • Language: Leave this on Java.
  • Spring Boot: Pick 3.0.0-SNAPSHOT (living on the edge!)
  • Project Metadata: Leave it be.
  • Dependencies: Spring Web and Mustache

With all this selection, click GENERATE. You’ll get a ZIP download. Unpack it and then import that new folder into your favorite IDE.

2 — Building a domain of knowledge

We need SOMETHING to put on it. Since this is Spring Boot 3 and we’re on Java 17, we can use records to define a nice simple type:

record Video(String name) {  
}

We’ll create a one-field record type that represents a YouTube video. (Why not?)

Java 17 records don’t require constructors, equals, hashCode, or toString methods. They come fully loaded. Yes, it’s possible to augment them, but for the simplest of POJOs, this is all it takes!

3 — Assembling a controller

Controllers are classes that process web requests. And so, to serve up a web template, we’ll need one!

Make a HomeController class like this:

@Controller  
public class HomeController {  
  
private List<Video> videos = List.*of*( *//* new Video("Need HELP with your SPRING BOOT 3 App?"), *//* new Video("Don't do THIS to your own CODE!"), *//* new Video("SECRETS to fix BROKEN CODE!"));
  @GetMapping("/")  
  public String index(Model model) {  
    model.addAttribute("videos", videos);  
    return "index";  
  }  
}

The @Controller annotation lets Spring Boot know that this class handles web requests and uses a templating engine.

The @GetMapping(“/”) annotation signals that this index() method will process HTTP GET / calls. Since we want to populate the template with data (well, canned data in this article), we have injected a Spring Model object.

We set attribute “videos” with the actual values from our little List of Video objects. Anything assigned to this model will be merged with the generated web page.

Finally, we return the name of the template we wish to render. Now Spring Boot will take this name and marry it with the templating engine of choice to actually find the template, which we’ll cover in the next section.

4 — Crafting a templating

Since we used Mustache, Spring Boot will expand the name “index” into src/main/resources/templates/index.mustache. Create a file of that name and add the following:

<h1>Greetings Spring Boot Learning readers!</h1>  
  
<ul>  
  {{#videos}}  
    <li>{{name}}</li>  
  {{/videos}}  
</ul>  
  
<form action="/new-video" method="post">  
  <input type="text" name="name">  
  <button type="submit">Submit</button>  
</form>

Mustache is incredibly lightweight. The {{#videos}} tag (notice the sideways mustaches?) is the directive to expand the list of the “videos” attribute into a separate

  • element for each entry in that list.

    Each entry is then accessible by property name. Given our Video object has a “name” field, we can reach each it using {{name}}.

    The rest is a simple HTML form with one text input for the “name” and a submit button.

    5 — Processing a new entry

    To consume this form, we need to go back to the HomeController and another web method handler, like this:

    @PostMapping("/new-video")  
public String newVideo(@ModelAttribute Video newVideo) {  
  List<Video> extend = new ArrayList<>(videos);  
  extend.add(newVideo);  
  this.videos = List.*copyOf*(extend);  
  return "redirect:/";  
}

    Spring Web’s @PostMapping annotation maps the HTTP POST /new-video call onto this method. And the @ModelAttribute will deserialize the incoming data from the HTML form into another Video object.

    Earlier in this article, we initialized videos using List.of(), which creates an immutable list. To expand it, we have to effectively replace it with a bigger one. And so create a new list, add the new entry, and then use List.copyOf() to produce a new, immutable one.

    Finally, with the new entry added, we can return back a “redirect:/”, which will tell the browser to redirect back to “/”, which should hit our first web method coded earlier, causing it to re-render the original template but with our new data.

    With all this in place, we can now run the class original generated by start.spring.io that has the @SpringBootApplication annotation applied and take our application for a spin.
