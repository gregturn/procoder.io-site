Spring Boot 3.0 approaches!

We’re getting into the home stretch. With Spring Boot 3.0.0-RC1 coming out last week, some wickedly cool stuff is headed your way.

1 — Jakarta EE 9

I recently created a video on exactly what Jakarta EE is in the grand scheme of Java, but the nugget to chew on is that Spring Boot 3.0 will be bearer of Jakarta EE 9.

This is the version where all the enterprise specs migrate from javax.* to jakarta.* .

Some specs are part of the Java SE and never were part of Java EE, so they are not part of Jakarta EE 9. This includes stuff like JDBC, which I flubbed my recent video by mentioning it as one of the standards that migrates. It does not. But JMS, servlets, JNDI, and most everything else is!

And it’s not just that the package names are moving. The latest releases for our most favorite packages will be there, including:

  • Apache Tomcat 10
  • Jetty 11
  • Hibernate ORM 6.1
  • ActiveMQ Artemis
  • more!

To keep up with the hottest, most popular Java packages out there, it’s time to pick up and run with Jakarta EE 9.

2 — Native applications on GraalVM

GraalVM is an alternative to the JVM and allows you to run “native” applications. This means faster start up times, more efficient memory usage, and other slick benefits.

To help you pick up and run with it, Spring Native was released as an extra package to ease usage with Spring Boot 2.7.

With Spring Boot 3.0, that support is built in. No Spring Native module required. And the rest of the Spring portfolio is also adopting the needed tweaks and adjustments to make Spring native-ready.

On top of that, the Spring team has been working hand-in-hand with Oracle, the purveyors of GraalVM, to ensure the entire experience is smooth and operational.

If you haven’t explored native applications, Spring Boot 3.0 may be the place to give it a go.

3 — Java 17 LTS

Spring Boot 3.0/Spring Framework 6.0 is moving the baseline of Java support up to the latest LTS (Long Term Support) version 17.

Now Spring has always been cautious to avoid forcing people to jump too soon to a newer version of Java. That’s why, in the past, there has been ample support for features found in newer versions of the JDK, without forcing their adoption.

But with Java 17 having gone GA in September of 2021, support from vendors like Oracle all the way into 2026 (and extended support off to 2029), Java 17 is a safe bet for your next Spring Boot application.

Java 17 also come chock full of some of the coolest features as well. Not only is it much more performant than Java 8, there are lots of cool features including:

  • Pattern matching for instanceof checks
  • Records
  • Sealed classes
  • Text blocks
  • Switch expressions
  • A huge bump in the various Collections operations (including things like List.of, Map.of, Set.of, etc.)
  • Pattern matching with switch operations (preview feature)

With this fully armed and operational JDK, who can say no?

4 — Project Loom arrives

Project Loom, through JEP-425, brings lightweight Virtual Threads as a new concurrency model to build applications on top of.

We’ve heard lots of rumors, promises, and even some strong back-and-forth. JEP-425 has landed Project Loom as a preview feature in Java 19.

Project Loom isn’t specifically a Spring Boot 3.0 feature. Instead, it’s an easy to grab thing since you’ll already be starting any new Spring Boot 3.0 application using Java 17. It’s as easy as sdk install java <your favorite vendor's v.19 JDK> to install, and simply bump java.versionto 19in your build file!

With that in place, you can easily start exploring. There is a LOT of hunger in the market for:

  • More efficient usage of existing resources.
  • Easier ways to scale applications.
  • Faster throughput on the whole.

We have been looking for silver bullets for years. Sadly, there are none. Nothing “magically” speeds anything up.

But there is no better time than now to evaluate which tactics are right for you. That’s why you should read Spring Data team leader Mark Paluch’s article Embracing Virtual Threads to get the long and short on what Project Loom ACTUALLY does, how it does it, and whether or not it will help your application scale or not.
